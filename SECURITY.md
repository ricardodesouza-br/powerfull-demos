# Security Policy

Thanks for taking the time to help keep **Powerfull Demos** safe.

> **Warning:** Security handling (secrets, connections, data exposure, compliance constraints) is **handled on a per-demo basis**. Always follow the specific guidance inside each demo folder’s `README.md`.

## What to report

Please report:

- Vulnerabilities or security issues in demo instructions, scripts, or artifacts
- Issues that could lead to credential leakage, unsafe configuration patterns, or data exposure

## How to report

Open a GitHub issue describing:

- A clear title (e.g., `Security: <short description>`)
- Steps to reproduce (if applicable)
- Impact/affected demo(s) (if known)
- Suggested remediation (if known)

## Response

We will review reports and update the relevant demo documentation or scripts as needed.
