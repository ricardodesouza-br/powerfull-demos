# Contributing

Thanks for your interest in improving **Powerfull Demos**! Contributions are welcome via **Pull Requests**.

## What to contribute

- New demos (add a folder under `Apps/`, `Agents/`, or `Automation/`)
- Improvements to existing demo documentation
- Fixes to demo scripts/instructions
- Updates to the curated demo list in the root `README.md` (optional but recommended)

## Demo folder requirements

For any new demo folder, include a `README.md` with:

- Demo summary (1–3 sentences)
- Setup requirements / prerequisites
- Step-by-step execution
- Link to the YouTube video for setup/execution
- Artifacts inventory (if any) and where they live

## Pull Request checklist

- [ ] Demo instructions are accurate and complete (setup + execution)
- [ ] Any scripts/artifacts mentioned in the README exist in the demo folder
- [ ] No secrets/credentials are committed
- [ ] The demo README includes the YouTube link
- [ ] The PR title/description clearly states what changed

## Notes

> **Warning:** Demo-specific prerequisites, assumptions, and security handling are handled per demo. PRs should ensure demo-local correctness and documentation quality.
